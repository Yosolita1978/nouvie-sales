## Learned
- `CustomerListItem` is reused as a generic customer shape (order edit form, customer detail cast). New list-only fields must be **optional** or those call sites break the build (caught by `tsc`).
- Prisma `groupBy` supports relation filters in `where` (e.g. `orderItem.groupBy({ where: { order: { orderDate: {...} } } })`), which keeps the top-products query in one call.
- The mobile bottom bar slices `navItems` as `[0,2)` + center + `[2,)`; a 5th item unbalances it, so report-only pages belong in the desktop sidebar only.
